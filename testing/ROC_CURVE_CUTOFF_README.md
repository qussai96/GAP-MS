
# ROC Curve and Score Cutoff Selection - Detailed Documentation

## Overview

This document explains the Receiver Operating Characteristic (ROC) curve implementation and the automatic score cutoff selection algorithm used in the GAP-MS pipeline. This process is used to determine optimal score thresholds for protein classification (supported vs. unsupported).

## Background Concepts

### What is a ROC Curve?

A ROC curve is a graphical representation used to evaluate the performance of a binary classifier at different classification thresholds. It plots:
- **X-axis (False Positive Rate - FPR)**: The proportion of unsupported proteins that are incorrectly classified as supported
- **Y-axis (True Positive Rate - TPR)**: The proportion of truly supported proteins that are correctly classified

Mathematically:
- **TPR (Sensitivity)** = TP / (TP + FN) = supported proteins that passed cutoff / all supported proteins
- **FPR (1 - Specificity)** = FP / (FP + TN) = unsupported proteins that passed cutoff / all unsupported proteins

An ideal classifier would have a curve that goes toward the top-left corner (high TPR, low FPR). The curve is generated by varying the classification threshold from high to low and computing TPR and FPR at each threshold.

### Why Use an Elbow Method?

Rather than using a single metric like AUC, this implementation uses an **elbow-based approach** to find the optimal cutoff. The key insight is:
- At very high cutoffs: Few proteins pass, but those that do are likely correct (high TPR, low FPR) → steep slope
- At low cutoffs: Many proteins pass, including many incorrect ones (lower TPR, high FPR) → shallow slope
- **Optimal point**: Where the slope transitions from steep to shallow (slope = 1) → best balance between sensitivity and specificity

## Implementation Details

### Input Data Structure

The algorithm operates on:
1. **`all_scores_df`**: DataFrame containing all proteins with their external scores
2. **`high_confident_proteins`**: Set of protein IDs that are known to be correct (validation set)
3. **`score_column`**: The name of the scoring column to evaluate (e.g., 'external_score')

### Algorithm Steps

#### Step 1: Pre-calculate Unsupported Proteins (Performance Optimization)
```python
# Pre-calculate unsupported scores once (expensive operation)
high_conf_set = set(high_confident_proteins_scores)
unsupported_scores = [s for s in scores if s not in high_conf_set]
```
- **Critical optimization**: Convert high-confidence list to set for O(1) lookup
- Calculate unsupported scores once instead of 100+ times
- Dramatically improves performance compared to filtering on each cutoff

#### Step 2: Generate Threshold Values
```python
cutoffs = np.linspace(1.0, 0.0, 100)
```
- Creates 100 evenly-spaced cutoff thresholds from 1.0 to 0.0
- Balanced between curve smoothness and performance
- Higher values create smoother curves but increase computation time

#### Step 3: Calculate Sensitivity (TPR) at Each Threshold
```python
def calculate_sensitivity(scores, cutoff):
    if not scores:
        return 0.0
    return sum(score >= cutoff for score in scores) / len(scores)
```
- For high-confidence proteins: What fraction passes each cutoff?
- As cutoff decreases: More proteins pass → TPR increases from 0 to 1
- Returns 0.0 if no high-confidence proteins (edge case)

#### Step 4: Calculate False Positive Rate (FPR) at Each Threshold
```python
def calculate_fpr(unsupported_scores, cutoff):
    if not unsupported_scores:
        return 0.0  # All proteins are high-confidence, FPR = 0
    unsupported_passing = sum(s >= cutoff for s in unsupported_scores)
    return unsupported_passing / len(unsupported_scores)
```
- **Bug fix**: Previously calculated precision instead of FPR
- Correct calculation: (unsupported proteins passing) / (all unsupported proteins)
- This ensures a proper concave ROC curve
- Reuses pre-calculated `unsupported_scores` from Step 1

#### Step 5: Sort Points by FPR
```python
roc_points = sorted(zip(fpr, tpr, cutoffs), key=lambda x: x[0])
fpr_sorted, tpr_sorted, cutoffs_sorted = zip(*roc_points)
fpr_sorted = list(fpr_sorted)
tpr_sorted = list(tpr_sorted)
cutoffs_sorted = list(cutoffs_sorted)
```
- **Critical step**: Without sorting, points are connected in arbitrary order → zigzagging curve
- After sorting by FPR: Creates smooth concave curve from (0,0) to (1,1)
- FPR is the x-axis → must be monotonically increasing for proper visualization
- Converting back to lists enables index-based iteration

#### Step 6: Find Optimal Cutoff (Elbow Detection)
```python
for i in range(2, len(tpr_sorted)):
    delta_fpr = fpr_sorted[i] - fpr_sorted[i - 1]
    slope = (tpr_sorted[i] - tpr_sorted[i - 1]) / delta_fpr if delta_fpr != 0 else float("inf")
    
    if slope < 1:
        cutoff = max(cutoffs_sorted[i], 0)
        return cutoff
```
- Iterates through sorted ROC points
- Calculates slope between consecutive points: `slope = ΔtprΔfpr`
- **When slope < 1**: The ROC curve begins to level off (diminishing returns)
- Returns the first cutoff where this occurs
- Ensures minimum cutoff of 0.5: `cutoff = max(cutoff, 0.5)`

### Visualization

The `plot_roc_curve()` function generates:
- **Purple line with markers**: The ROC curve showing sensitivity vs. specificity across all thresholds
- **Red vertical line**: The selected optimal cutoff point
- **Red scatter point**: Exact location on the curve where cutoff is selected
- **Text annotation**: Displays the selected cutoff score value

Key plot improvements:

## Data Flow in the Pipeline

```
Input: all_scores_df, high_confident_proteins, score_column
         ↓
    find_cutoff()
         ↓
    Extract scores and identify unsupported proteins (one-time calculation)
         ↓
    Generate 100 cutoff thresholds (1.0 → 0.0)
         ↓
    Calculate TPR at each threshold using pre-computed high_confident_proteins_scores
         ↓
    Calculate FPR at each threshold using pre-computed unsupported_scores
         ↓
    Sort by FPR (critical for proper concave curve shape)
         ↓
    Find elbow point where slope < 1 (first occurrence)
         ↓
    Generate ROC plot with marked cutoff
         ↓
    Return: optimal_cutoff_score
         ↓
    find_apply_score_filter()
         ↓
    Ensure minimum cutoff of 0.5
         ↓
    Filter proteins: keep only those with score > cutoff
         ↓
    Output: cutoff value, set of supported proteins
```

## Performance Considerations

### Optimization Strategy
The algorithm uses several optimizations for speed:

1. **Set-based lookups**: High-confidence proteins converted to set for O(1) membership tests
2. **Pre-computation**: Unsupported scores calculated once outside loop, not 100+ times
3. **Balanced resolution**: 100 cutoff points provides smooth curves with reasonable performance

### Previous Issues (Now Fixed)
- **Before**: Recalculating unsupported scores for each cutoff → O(n²) complexity
- **After**: Pre-calculate once, reuse → O(n) complexity
- **Result**: 100x+ faster execution on large datasets

### Computational Complexity
- **Time Complexity**: O(n × m) where n = number of proteins, m = number of cutoffs (100)
- **Space Complexity**: O(n) for storing scores and subsets

## Interpreting Results

### Example from your data:
- **Score cutoff selected: 0.74**
- **Meaning**: 
  - Proteins with external_score > 0.74 are classified as supported
  - At this point, the ROC curve slope = 1 (elbow)
  - Above this cutoff: high TPR but rising FPR (diminishing returns)
  - Below this cutoff: small gains in TPR but large increase in FPR

### ROC Curve Shape Interpretation:
- **Perfect curve** (concave, top-left corner): Perfect classifier
- **Diagonal line** (FPR = TPR): Random classifier (useless)
- **Your curve**: Should be a smooth concave shape
  - Steep at beginning: Good discrimination at high scores
  - Levels off: Lower scores provide less discrimination
  - Elbow point: Optimal balance between specificity and sensitivity

## Bug Fixes and Improvements

### Bug #1: Incorrect FPR Calculation (FIXED)
- **Symptom**: ROC curve doesn't have proper concave shape
- **Root Cause**: Was calculating precision instead of FPR using: `(high_conf_passing) / (all_passing)`
- **Fix**: Now correctly calculates: `(unsupported_passing) / (all_unsupported)`
- **Impact**: ROC curves now display properly with characteristic bow toward top-left

### Bug #2: Points Not Sorted by FPR (FIXED)
- **Symptom**: Zigzagging curve with FPR=1 at TPR=0
- **Root Cause**: Points connected in order of cutoff iteration, not sorted by FPR
- **Fix**: Sort all FPR/TPR/cutoff tuples by FPR value before plotting
- **Impact**: Smooth concave curves instead of chaotic zigzags

### Performance Issue: Slow Execution (FIXED)
- **Symptom**: Function takes much longer after FPR calculation fix
- **Root Cause**: Recalculating unsupported scores for each cutoff (100+ times)
- **Fix**: Pre-calculate unsupported_scores once using set-based lookup
- **Impact**: ~100x faster execution, now comparable to original speed

## Common Issues and Troubleshooting

### Issue: Cutoff not found or unexpected value
- **Cause**: Slope never drops below 1 (rare) or high-confidence set too small
- **Solution**: Returns 0 as fallback, then enforced minimum of 0.5 in `find_apply_score_filter()`
- **Check**: Verify high-confidence protein set is defined correctly

### Issue: Too many/few proteins passing filter
- **Cause**: Cutoff value inappropriate for your data or elbow not well-defined
- **Solution**: Check the generated ROC curve plot visually; ensure elbow point is reasonable
- **Action**: May need to adjust slope threshold `if slope < 1:` or minimum cutoff value

## Mathematical Formulation

### Slope Calculation at Each Point
$$\text{slope}_i = \frac{\Delta \text{TPR}}{\Delta \text{FPR}} = \frac{\text{TPR}_i - \text{TPR}_{i-1}}{\text{FPR}_i - \text{FPR}_{i-1}}$$

### Optimal Cutoff Selection
$$\text{Find } i \text{ where } \text{slope}_i < 1 \text{ (first occurrence)}$$
$$\text{cutoff}_{\text{optimal}} = \max(\text{cutoff}_i, 0.5)$$

### Sensitivity (TPR)
$$\text{TPR} = \frac{\text{High-confidence proteins passing cutoff}}{\text{All high-confidence proteins}}$$

### False Positive Rate (FPR) - CORRECTED
$$\text{FPR} = \frac{\text{Unsupported proteins passing cutoff}}{\text{All unsupported proteins}}$$

**Note**: The correct formula uses unsupported (negative) proteins, not all proteins passing the cutoff.

**Previous Bug**: The original implementation incorrectly calculated this as precision:
$$\text{Wrong: } \frac{\text{High-confidence proteins passing}}{\text{All proteins passing}}$$
This caused incorrect curve shapes. The fix ensures proper ROC curve geometry.
